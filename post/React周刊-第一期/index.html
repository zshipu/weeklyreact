Remix 中关于服务器状态的一些思考

关于 Remix 中状态和服务器调用管理的一个有趣想法，它被设计为在客户端使用和不使用 JavaScript。它可以回退到普通的旧浏览器行为，而 JS 只是一个奖励。

在同一台服务器上运行 Next 和 Remix

一个有趣的设置，它表明我们可以在同一个 Node.js 服务器中混合使用 Remix 和 Next.js。这有助于设计一个增量迁移策略来采用或评估 Remix。

Remix、SQLite 和 Prisma， 神奇！

另一个很好的 Remix 功能演示。能够从加载器函数调用 Prisma 是非常强大的。它提醒其他解决方案，例如 Next.js（但以更集中的方式）和服务器组件。

移动 2.6

此 React 视频制作库的最新版本。Remotion 播放器现在很稳定。新的错误叠加和听力图模板。下一个版本肯定会是 3.0，具有 AWS lambda 无服务器渲染支持和快速编码视频的能力！

附加功能：

  • 避免 Remix 加载器中的查询瀑布：即使 Remix 设法并行化请求，您仍然必须 Promise.all () 在加载器函数中使用
  • Remix 正在招募：远程优先，最少 4 小时与美国重叠
  • Preact 现在有一个教程：10 分钟来学习这个 React 轻量级替代方案，并带有一个不错的代码编辑器广告预览
  • Relay 13.0：包括新的 Rust 编译器
  • Nextra 2.0 work-in-progress：为 Next.js SSG (docs/blog) 工具的这次升级计划了很好的堆栈。Docusaurus 的良好竞争😯
  • 创建 3D 粒子效果的三种方法：对于像我这样学习 Three.js 的人😏

React Native 工程师构建 SwiftUI 应用程序

一个尝试 SwiftUI 的 React-Native 开发者。React 及其声明性模型的知识是可重用的。对动画、导航和 SwiftUI “热重载”的积极反馈。关于错误消息、嵌套语法和其他一些细节的混合反馈。

附加功能：

  • react-native-reanimated-carousel : 基于 Reanimated 2 的新现代轮播库，实际维护
  • Marc Rousavy 使用 react-native-jsi-image将我们升级到 JSI 并且 “ react-native-camera 现在已被弃用，取而代之的是 react-native-vision-camera”
  • Fela：为 React 和 React Native 编写状态驱动的样式
  • React Native 中所有可用键盘类型的概述：方便的备忘单资源
  • App.js conf 六月回归，CFP 明天开启
  • docusaurus-react-native-plugin : React-Native-Web 支持 😍
  • 🎥 The Matrix Reacts ：William Candillon 的第一个 React-Native-Skia 教程，带有很棒的 Remotion 介绍
  • 🎙️ RNR 223 - React.js 和 React Native 的区别？
